# 单元测试设计文档

## 测试策略

如果后续开始实现外部 PostgreSQL/MySQL 迁移，需要新增一组跨 backend 契约测试。测试目标不是只测 SQL 能跑，而是验证 SQLite/PG/MySQL 在业务语义上等价。

## 单元测试设计

### UT-001 MetadataStore 契约测试

- 覆盖：`record_async_job`、`list_async_jobs`、`get_async_job_record`、`summarize_async_jobs`。
- 断言：状态、排序、过滤、分页、隐藏任务、错误信息、result_json 都一致。
- 后端矩阵：SQLite、PostgreSQL；MySQL 在第二阶段加入。

### UT-002 Gallery/Waterfall 契约测试

- 覆盖：画廊记录写入、瀑布墙记录写入、排序、过滤、推荐/置顶/屏蔽状态更新。
- 断言：同一 job 多图、局部图片结果、隐藏 gallery/wall 的行为一致。

### UT-003 ImageConversation 契约测试

- 覆盖：保存、替换、删除、清空、跨 API key 可见性。
- 断言：历史记录不因账号/API key 删除而不可见。

### UT-004 AccountStore 契约测试

- 覆盖：添加账号、删除账号、刷新状态、成功/失败计数、图片配额状态。
- 断言：并发刷新不会全量覆盖掉其他写入。

### UT-005 APIKeyStore 契约测试

- 覆盖：创建、更新、旋转、删除、鉴权、请求配额、图片配额。
- 断言：配额扣减必须具备事务语义；并发扣减不超卖。

### UT-006 JobStore 契约测试

- 覆盖：提交任务、更新任务、partial result、完成/失败、读取结果和日志索引。
- 断言：没有本地 result JSON 时仍能从 DB 返回任务结果。

### UT-007 SettingsStore 契约测试

- 覆盖：配置保存、环境变量覆盖、DB 配置覆盖、敏感字段处理。
- 断言：重启后设置仍存在，环境变量优先级稳定。

## 集成测试设计

### IT-001 单实例 SQLite 回归

- 目标：默认行为不变。
- 场景：不设置外部 DB，完整跑现有异步任务、OpenAI 兼容图片跟踪、画廊、瀑布墙、配置保存、API key 管理。

### IT-002 PostgreSQL 单实例

- 目标：所有元数据和 JSON 源迁入 PG 后功能可用。
- 场景：启动、创建 API key、添加账号、提交异步图片、流式 partial result、查看画廊/瀑布墙/任务列表/会话记录。

### IT-003 PostgreSQL 多实例

- 目标：多容器共享同一个外部 DB，不依赖实例本地任务 JSON。
- 场景：A 实例提交任务，B 实例查看任务/画廊/瀑布墙/日志索引。

### IT-004 SQLite -> PostgreSQL 迁移 CLI

- 目标：旧数据可迁移。
- 校验：任务数、画廊图片数、瀑布墙图片数、会话数、API key 数、账号数、配置项、图片 URL/path 抽样一致。

### IT-005 MySQL backend

- 目标：PG 稳定后验证 MySQL。
- 重点：upsert、JSON、排序规则、事务隔离、并发配额扣减。

## 属性测试设计

### PT-001 排序和分页稳定性

- 随机生成不同 `created_at`、`updated_at`、状态、模型、提示词数据。
- 验证不同 backend 的排序、分页、过滤结果 ID 序列一致。

### PT-002 并发配额扣减不超卖

- 随机并发请求数和配额上限。
- 验证成功扣减次数不超过配额，失败请求不污染计数。

### PT-003 任务状态状态机

- 随机生成 queued/running/succeeded/failed 转换序列。
- 验证非法转换被拒绝或不会破坏最终可读状态。

### PT-004 JSON payload 往返

- 随机生成包含中文、长提示词、数组、嵌套对象、空值的 payload。
- 验证写入再读出不丢字段、不乱码、不改变类型。
