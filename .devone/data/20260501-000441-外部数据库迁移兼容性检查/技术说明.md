## 关键发现

1. `services/metadata_db.py` 是 SQLite 专用实现，不是可插拔数据库抽象。
2. 外部 DB 不能只替换 metadata DB，因为配置、账号、API key、CPA/Sub2API 配置仍以 JSON 为真实源。
3. 异步任务运行仍依赖 `jobs/*.json`、`job_results/*.json`、`task_logs/*.log`。
4. 图片实体应继续由本地文件、HF datasets 或未来对象存储负责，DB 只保存元数据。
5. PostgreSQL 比 MySQL 更适合作为第一阶段外部 DB backend。

## 影响面

- 后端服务：`metadata_db`、`job_service`、`config`、`account_service`、`api_key_service`、`cpa_service`、`sub2api_service`、`data_service`。
- API：异步任务接口、OpenAI 兼容图片接口、画廊、瀑布墙、设置、账号/API key 管理。
- 前端：任务列表、画廊、瀑布墙、设置页依赖后端持久化语义；前端本身不是迁移主战场。
- 部署：单实例持久卷可以继续工作；多实例/无状态容器需要外部 DB + 外部图片存储。
