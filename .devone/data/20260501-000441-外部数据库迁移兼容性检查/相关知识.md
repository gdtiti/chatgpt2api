# 相关知识

## 本轮证据摘要

### `services/metadata_db.py`

- 主要 SQLite 元数据层，默认路径为 `DATA_DIR / "metadata.sqlite3"`。
- 初始化表：
  - `settings_snapshots`
  - `accounts`
  - `async_jobs`
  - `gallery_images`
  - `image_conversations`
  - `task_logs`
  - `system_files`
  - `request_logs`
- 典型 SQLite 绑定：`sqlite3`、`sqlite3.Row`、`PRAGMA quick_check`、`PRAGMA journal_mode=WAL`、`sqlite_master`、`PRAGMA table_info`、`INTEGER PRIMARY KEY AUTOINCREMENT`、`?` 参数占位符、本地 corrupt 文件隔离。
- `record_async_job()` 会写 `async_jobs`，再调用 `_record_gallery_images_locked()` 同步画廊/瀑布墙图片元数据。
- `list_async_jobs()`、`list_gallery_jobs()`、`list_waterfall_images()` 已按排序/过滤查询 DB，但仍是 SQLite SQL。
- `list_image_conversations()` 当前查询不按 `api_key_id` 过滤；`upsert_image_conversation()` 的主键仍是 `(conversation_id, api_key_id)`。

### `services/job_service.py`

- `JobService.__init__()` 传入 `jobs_dir`、`job_results_dir`、`task_logs_dir`，并根据 `jobs_dir.parent / "metadata.sqlite3"` 选择 metadata DB。
- `_job_file()`、`_result_file()`、`_task_log_file()` 构造本地文件路径。
- `_read_json()`、`_write_json()` 是任务和结果文件的真实读写函数。
- `submit_job()` 先写 job JSON，再记录 metadata，再交给线程池执行。
- `_store_partial_image_result()` 对流式/批量图片结果写 result JSON，并同步 metadata。
- `list_gallery_jobs()`、`list_waterfall_images()` 主要从 metadata DB 查询。
- `get_job_result()` 需要结合 metadata 记录和 result 文件。

### `services/config.py`

- 支持 `CHATGPT2API_DATA_DIR` 和 `CHATGPT2API_CONFIG_FILE`。
- 默认容器内数据目录是 `/app/data`，配置文件默认 `/app/data/config.json`。
- `ConfigStore._save()` 写 `config.json`，然后 `metadata_db.record_settings()` 记录快照。
- `api_keys_file`、`accounts_file`、`jobs_dir`、`job_results_dir`、`task_logs_dir` 都从 `DATA_DIR` 派生。

### `services/account_service.py`

- `AccountService(config.accounts_file)`。
- `_load_accounts()` 从 `accounts.json` 读取。
- `_save_accounts()` 写 `accounts.json`，然后调用 `metadata_db.record_accounts()` 镜像。
- 因此 DB 中 `accounts` 不是账号真实源。

### `services/api_key_service.py`

- `APIKeyService(config.api_keys_file)`。
- `_load_items()` / `_save_items()` 读写 `api_keys.json`。
- API key 的 hash、权限、配额、计数等仍在 JSON 中。
- 当前 metadata DB 没有 API key 表。

### `services/cpa_service.py` 与 `services/sub2api_service.py`

- `cpa_config.json` 保存 CPA pool 与导入 job。
- `sub2api_config.json` 保存 Sub2API server 与导入 job。
- 导入任务状态通过配置 JSON 更新，不在 metadata DB。

### `services/data_service.py`

- `save_image_bytes()` 根据 `config.image_storage_backend` 选择本地文件或 HF datasets。
- 本地模式写图片、缩略图、瀑布墙图。
- HF datasets 模式上传三类文件到数据集路径。
- 返回 `relative_path`、`thumbnail_relative_path`、`wall_relative_path`、URL 和 markdown。
- `DataMaintenanceService.collect_stats()` / `cleanup()` 依赖本地目录遍历。

### `api/app.py`

- 启动时 `metadata_db.record_settings(config.get())`、`metadata_db.record_accounts(account_service.list_accounts())`。
- HTTP 中间件写 `request_logs`，失败被吞掉，不影响请求主流程。

### `api/ai.py`

- `_openai_compat_image_tracking_options()` 读取是否创建任务、纳入画廊、纳入瀑布墙。
- `/v1/images/generations`、`/v1/images/edits` 在非流式和流式场景都会创建 inline job 并通过 `finish_inline_job()`/`fail_inline_job()` 同步任务状态。

### 测试现状

- `test/test_metadata_db.py` 直接依赖 SQLite 行为，包括 corrupt quarantine、`sqlite_master`、`metadata.sqlite3`。
- `test/test_async_jobs.py` 大量通过临时目录、`jobs`、`job_results`、`task_logs` 和 `metadata_db` 验证异步任务、画廊、瀑布墙、会话。
- `test/test_api_keys.py` 仍以 `api_keys.json` 构造 API key service。

## 稳定事实

- 当前项目可以通过持久化 `/app/data` 保证单实例容器重启不丢数据。
- 当前项目不能通过只配置外部 DB 完成全功能迁移，因为没有外部 DB backend 和统一 Store 抽象。
- 迁移到外部 DB 的重点不是 schema，而是把“真实源”从本地 JSON/文件迁移出去。
