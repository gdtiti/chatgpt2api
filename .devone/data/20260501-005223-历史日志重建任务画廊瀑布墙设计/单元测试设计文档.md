# 单元测试设计文档

## 主要被测行为

- 日志行解析和事件合并。
- 图片目录扫描和原图/缩略图/瀑布墙图配对。
- job/result JSON 扫描和 result data 抽取。
- metadata 恢复写入的幂等性和状态保留。
- 前端页面大小选择和恢复报告展示。

## 必须覆盖的失败模式

- 截断 JSON 日志行。
- 重复事件。
- 同一 job 多个来源互相补充。
- task log 路径不存在。
- 缩略图存在但原图不存在。
- 原图存在但 thumbnail/wall 不存在。
- 当前 DB 已有 gallery_images，恢复不能覆盖置顶/屏蔽状态。
- system.log 只有 job_id/status，没有 prompt。

## 单元测试设计

### UT-001 日志解析

- 输入：合法结构化日志、普通文本日志、截断 JSON。
- 断言：合法行生成事件；坏行进入 warnings；扫描不中断。

### UT-002 事件合并

- 输入：submitted -> started -> succeeded / failed 多种顺序。
- 断言：同一 job_id 合并成一个候选；最终状态按优先级确定；created_at/updated_at 正确。

### UT-003 图片目录扫描

- 输入：原图、`-thumb`、`-wall` 混合文件。
- 断言：只把原图作为候选；正确配对 thumbnail/wall；不重复导入缩略图。

### UT-004 job/result JSON 扫描

- 输入：job JSON 和 result JSON。
- 断言：恢复 payload、prompt、result、preview_images。

### UT-005 metadata 旧库扫描

- 输入：临时 SQLite 旧库。
- 断言：读出 async_jobs/gallery_images；坏库只产生 warning。

### UT-006 apply 幂等

- 连续 apply 同一 report 两次。
- 断言：第二次新增数为 0；总数不重复增长。

### UT-007 preserve_state

- 当前 gallery image 已置顶/屏蔽。
- 恢复同一 job/image。
- 断言：`is_pinned`、`is_blocked` 保持原值。

## 集成测试设计

### IT-001 从空 metadata + images 目录恢复画廊

- 构造 images 目录和空 DB。
- scan -> apply。
- 断言 `/api/gallery`、`/api/gallery/wall` 可看到恢复图片。

### IT-002 从 system.log 恢复任务列表

- 构造 system.log 的 async/inline job 事件。
- scan -> apply。
- 断言 `/api/async/jobs` 可看到 recovered job，状态正确。

### IT-003 混合来源恢复

- 同一 job 同时存在 log 和 image 文件。
- 断言 job 与 image 关联到同一 job_id，不产生 synthetic duplicate。

### IT-004 前端分页大小

- 画廊构造超过 20 条。
- 断言页面能选择 50/100，并显示 total。

## 属性测试设计

### PT-001 日志事件顺序随机化

随机打乱同一 job 的事件顺序，验证最终合并结果稳定。

### PT-002 文件名随机化

随机生成 request_id、index、thumb/wall 组合，验证扫描器不会把派生图当原图。

### PT-003 幂等 apply

随机生成候选集合，多次 apply 后数据库唯一键数量保持稳定。

## 禁止使用的捷径

- 不允许用假 prompt 填充缺失提示词。
- 不允许清空 gallery 后重建。
- 不允许扫描任意系统路径。
- 不允许把解析失败当作成功。
