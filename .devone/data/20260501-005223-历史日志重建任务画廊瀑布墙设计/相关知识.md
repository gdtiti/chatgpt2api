### 现有恢复逻辑

`services/job_service.py` 有 `_backfill_metadata_if_empty()`：

- 只有当 `metadata_db.has_any_async_job_records(...)` 为 false 时才回填。
- 只从 `jobs/*.json` 扫描，最多 500。
- 当前 `data/jobs` 为 0，所以此逻辑无法恢复本地样本中的画廊/瀑布墙。
- 如果 metadata 已经有少量任务但 gallery 为空，也不会触发补齐。

### 图片保存规则

`services/data_service.py.save_image_bytes()` 写入：

- 原图：`DATA_DIR/images/<YYYY-MM-DD>/<request_id>-<index>.<ext>`
- 缩略图：`<request_id>-<index>-thumb.<ext>`
- 瀑布墙图：`<request_id>-<index>-wall.<ext>`

返回 metadata：`relative_path`、`thumbnail_relative_path`、`wall_relative_path`、`url`、`thumbnail_url`、`wall_url`、`markdown`。

### 日志图片保护

`utils/log.py` 会把日志中的 base64 图片保存成文件，然后把日志里的 base64 替换成 `[image: <url>]`。这意味着：

- 历史日志可能保存了图片 URL，而不是原始 base64。
- 如果 `images` 目录仍在，可以通过这些 URL/path 反推画廊项。
- 如果图片实体已丢，只靠日志不能恢复图片内容。

## 可恢复性分级

```text
等级       来源组合                                      可恢复内容
高可信     metadata.sqlite3 / 旧 metadata.sqlite3          原任务、画廊、瀑布墙、会话、请求日志
高可信     jobs/*.json + job_results/*.json               payload、prompt、结果、图片索引
中可信     task_logs/*.log 结构化事件 + 图片 URL           任务状态、部分 prompt、部分结果图片
中低可信   system.log 结构化任务事件                       job_id、状态、模型、api_key、log_path、时间
中低可信   images 目录扫描                                 图片索引、时间、派生图路径；prompt/账号缺失
不可恢复   已删除图片 bytes 且无远端 URL                    图片内容不可恢复
不可恢复   日志和 JSON 中从未记录的完整 prompt             不能凭空恢复
```
